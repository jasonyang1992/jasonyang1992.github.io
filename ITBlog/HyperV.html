Hyper-V 2016 Standalone

  1. Home
  2. IT Blog
  3. Hyper-V 2016

Posted on September 25, 2017 at 1:43 PM

The goal of this project was to understand how we can incorporate this operating system and create a completely virtualized environment like VMware ESXI environment.

When I first used Hyper-V in Windows Server 2012 R2, I thought it was amazing. I could create multiple machines and run them sententiously. It was so easy to maintain the computers in this environment. I then learned from reading articles on Spiceworks about Microsoft Hyper-V 2016 standalone. It sounded interesting because I wanted to know the difference between the Hyper-V in Windows Server 2012 R2 and the standalone..

I took a chance to install Hyper-V 2016 into an old computer just to see how it would work. I was very surprised after the installation because there was no GUI and it showed a command line prompt. It was intimating at first but thankfully we do have Google to research how to access Hyper-V.

I then learned that to access Hyper-V manager in Hyper-V 2016 standalone was using Windows Hyper-V manager from operating systems like Windows 8 or higher. You will also need to add credentials into both the Hyper-V 2016 standalone and the local machine. In my case, I used Windows 10 pro to access the Hyper-V manager (Note It must be a Windows Pro or higher to install Hyper-V manager, not home).

Here is the source that helped me figure this out

On the Hyper-V server


  1. Download Hyper-V 2016 Iso then use rufus to store it in bootable USB.
  2. Boot the USB on the computer and install it like how you would like any other oerpating system.
  3. Set the password, change the computer name, enable remote desktop access, and give it a static IP
  4. In the command prompt > type powershell > type Start-Process PowerShell â€“Verb RunAs
  5. type Enable-PSRemoting > Enable-WSManCredSSP -Role server

On The Windows 10 Pro Machine


  1. Change your network adapter setting, Click Start > Search "Homegroup" > Launch "Homegroup"
  2. Locate and edit "C:\Windows\System32\drivers\etc\hosts" as an administrator in notepad then add an entry like "192.168.0.2 (IP address of the Hyper-V) server-1(Name of the Hyper-V)"
  3. In powershell enter the following
    1. Set-Item WSMan:\localhost\Client\TrustedHosts -Value server-1(Name of the Hyper=V)
    2. Enable-WSManCredSSP -Role client -DelegateComputer server-1
    3. Enable-WindowsOptionalFeature -Online -FeatureName:Microsoft-Hyper-V -All
    4. cmdkey /add:server-1 /user:Administrator /pass
